# Enterprise Geospatial Data Management

## Overview
This project demonstrates enterprise geodatabase design and spatial data management for hydrographic datasets using ArcGIS Pro and SQL Server.

The goal was to build a structured geospatial database capable of managing large marine datasets while maintaining spatial integrity and enabling multi-user access.

## Objectives
- Design an enterprise geodatabase schema
- Organize hydrographic datasets into feature datasets
- Implement domains and topology rules
- Manage raster datasets using mosaic datasets

## Study Area
St. John’s Harbour, Newfoundland.

## Tools
- ArcGIS Pro
- Microsoft SQL Server
- ArcGIS Geodatabase

## Data Managed
- Survey boundaries
- Tracklines
- Grab samples
- Shipwrecks
- Satellite imagery
- Topographic features

## Workflow
1. Create enterprise geodatabase in SQL Server
2. Define feature datasets
3. Import spatial data layers
4. Implement attribute domains
5. Create raster mosaic dataset
6. Generate raster overviews

## Results
The project produced a structured geospatial database supporting hydrographic survey planning and spatial analysis.
